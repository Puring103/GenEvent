现在要完成项目的代码生成部分

# 生成目标
1. 为每个event实现publisher,文件名为{eventname}Publisher
2. 为每个有Ongameevent的类实现SubscriberRegistry，文件名为{subscribername}SubscriberRegistry
3. 将publish注册到EventPublishers中，将SubscriberRegistry注册到SubscriberRegistrys中

# 生成过程
1. 反射寻找所有event,以及所有附加了ongenevent属性的方法及其对应的类
2. 为每个订阅者类生成SubscriberRegistry，根据方法名注册委托，并在startlistening和stoplistening中添加对应事件的方法
3. 寻找所有使用了event作为参数的方法，实现publisher，按照ongameevent中指定的优先级按顺序调用各个订阅者
4. 在SubscriberRegistrys和EventPublishers中添加生成的类

# 实现细节
1. 使用unity的api进行反射，速度更快
2. 尽量不要使用appendline,使用字符串替换，模板放在单独的文件中
3. 代码生成器对外提供api，支持输入路径参数,供编辑器类调用
4. 实现一个简单的编辑器类，添加一个menu，在默认的路径生成代码
5. 为每个环节添加报错，输出出来，有报错则取消生成
6. ongenevent修饰的方法必须是public的，一个类订阅同一种event的方法只能有一个
